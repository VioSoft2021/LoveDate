WHAT TO ADD AT LAUNCH (NOT NOW):

  At ~500 users:
    - Real domain (lovedate.app or similar)  $10–15/year
    - Email service (Postmark/Resend)        $10–30/mo (optional)
    - Analytics (Plausible or PostHog)       $9–20/mo
    - Sentry / error monitoring              $26/mo

  At ~5,000 users:
    - Supabase Team upgrade                  $599/mo (from $25!)
    - Cloudflare Pro (DDoS + edge cache)     $20/mo
    - Backups beyond Supabase defaults       varies

  If going native apps:
    - Google Play Developer                  $25 one-time
    - Apple Developer                        $99/year

DANGER ZONES (cost spikes can hide here):

  1. Anthropic without a cap — single buggy loop could burn through
     card balance in hours. Cap is set to $25/mo. KEEP IT.
  2. Supabase Storage — photos at 5MB each fill 100GB Pro quota at
     ~20K photos. Image compression on upload cuts this 80%+.
     NOT YET SHIPPED. MED-priority for ~1K users.
  3. Realtime subscription leaks — CRIT-3 was verified false but
     the category of bug is real. Watch for it.
  4. Edge Function invocation multiplication — every page-load
     fires SW update checks, AI moderation, profile sync. Adds up
     at scale.

COST-CONTROL MILESTONES:

  Today:               Anthropic cap = $25/mo. Supabase Pro is the
                       ONLY paid Supabase subscription.
  Before 100 users:    Ship image compression in photo upload.
  Before 500 users:    Add per-user daily AI quota table (Phase F
                       item that's been deferred). Prevents one
                       heavy user from eating the AI budget.
  Before 5,000 users:  Enable prompt caching (HIGH-9, deferred).
                       Cuts AI input costs ~90%.

WHAT NOT TO PAY FOR YET:
  - Sentry (Crash Inbox covers it)
  - A real domain (github.io is fine until launch)
  - Email service (Supabase handles auth emails)
  - Analytics (premature without users to analyze)
  - Push provider (VAPID + pg_net works)

HONEST TAKE:
  Current $30–50/mo is sustainable for YEARS of friend testing.
  The jump that matters is at ~5,000 users when Supabase Pro stops
  being enough — that's when revenue needs to cover the $599/mo
  bump.

5. REBRAND + WOW FEATURE DIRECTION (decided 2026-05-21 morning)
================================================================

CONTEXT: Master committed to a name change after seeing the Play
Store collision (5M-download "LoveDate - AI Romantic Match" + 5
other apps). New name + new product direction needed.

DESIGN TARGET: a demanding 28-year-old in Bucharest who's tired
of dating apps but still hopes one will work. Name has to pass
the "dinner test" (would I tell a friend "I met them on ___"
without cringing?).

FIRST-ROUND NAME CANDIDATES (top 5, given for honest tradeoff):

  Name   | Meaning                       | Why it works
  -------+-------------------------------+-----------------------------
  Spera  | "to hope" (RO + LAT)          | Warm verb, active, present-
         |                               | tense ("I hope"). Romanian
         |                               | dinner test passes (people
         |                               | already say "sper că...").
         |                               | Almost certainly available.
  Anima  | "soul" (LAT/RO)               | Premium, depth-signal,
         |                               | beautifully short. Watch for
         |                               | "anime" confusion.
  Mira   | "wonder/marvel" (LAT + RO     | 4 letters, pronounceable
         | "to be amazed")               | everywhere. Verify Microsoft's
         |                               | Mira product is defunct.
  Codé   | "code" with French accent —   | Tech-savvy, ties to the
         | references personality DNA    | personality-code DNA. Accent
         |                               | in URLs is friction.
  Două   | "two/both" (RO)               | Most romantic concept in any
         |                               | language. Deeply local. But
         |                               | the ă kills SEO + i18n.

THEO'S RECOMMENDATION (informed pick, not final): SPERA.

PART 2 — IN-APP UX UPGRADES (no competitor does these well):

  1. Conversational onboarding
     Profile creation = 1 paragraph → AI fills 12 fields.
     Uses existing profile-writer infrastructure.
  2. Voice intro replaces opener
     20-second voice card in the deck. Plays muted by default.
     Reveals nervousness, warmth, accent — catfish-killer.
     Chat audio infra already exists.
  3. Conversation health check
     After 10 messages exchanged, AI shows BOTH users a
     transparent report: questions asked/answered, emotional
     reciprocity, future-orientation. Not gamification —
     accountability against ghosting.
  4. Soft preferences (not hard filters)
     "I prefer outdoorsy types" = ranking signal, not exclude.
     Deck reshuffles, doesn't shrink. Boo/Bumble do hard
     filters; nobody does soft.

PART 3 — THE "WOW" FEATURE (3 picks, descending risk):

  IDEA A — "Try a chat" preview (LOW risk, HIGH wow)
    Before swiping right, tap "what would we talk about?".
    AI runs a 3-turn simulated conversation between the two
    profiles right in the card. 5 seconds, you see the vibe.
    Reduces wasted matches. NO MAJOR APP DOES THIS.
    Built on existing Sonnet scoring infrastructure.

  IDEA B — Slow Match Mode (LOW risk, MEDIUM wow, brand-defining)
    Opt-in mode: one profile per day, 24h to send a real first
    message. Forces quality over quantity. Premium tier
    ("Spera Slow"). Counter-cultural — 25–34s sick of swipe
    fatigue will love it. Sapio app tried once; market is open.

  IDEA C — AI Twin Preview (HIGH risk, EXTREME wow)
    Each user has an AI version of themselves trained on
    bio + first chat messages. Before mutual match, both can
    have a 60-second simulated conversation with the other
    person's AI twin. Not their words — their vibe.
    Ethics need thought (consent, accuracy, misrepresentation
    risk). Mitigation: opt-in both sides, only stated
    preferences + writing samples, BIG "AI Twin Preview —
    not real words" label.
    BOO/TINDER/BUMBLE COULDN'T SHIP THIS — personality data
    too thin. Maps directly to existing AI Match Scoring +
    Profile Writer infrastructure.

PLAN (Master's order, 2026-05-21):
  1. Pick the new name (Master will see expanded list next).
  2. Execute full rebrand across codebase + config + memory +
     repo.
  3. Build Part 2 (UX upgrades) + Part 3 Ideas A and C.
     Theo to draft a roadmap when name is chosen.

DELIBERATELY DEFERRED:
  - Idea B (Slow Match Mode) — would clash with the swipe-deck
    UX the user already knows; revisit later as a premium tier.
